Fix post content not loading when reopening saved posts

Root cause: loadQuillContent set quill.root.innerHTML directly, which
bypasses Quill's internal Delta model. When Quill later tried to read
or manipulate the content, it didn't recognize it, causing the editor
to appear empty.

Fix:
- Use quill.clipboard.convert({ html }) to parse HTML into Quill's Delta format
- Use quill.setContents(delta, SILENT) to properly load it into the editor
- Add fallback: set innerHTML + quill.update() if convert fails
- Add error toast if post data fails to load from API
- Clear editor before loading new content
